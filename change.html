Change Control & Configuration Management

Managing change requests, baselines, and configuration control

📘 PMBOK

Integrated Change Control ensures all changes go through a Change Control Board. Configuration management maintains deliverable integrity, baselines, and documentation.

📗 PRINCE2

Configuration and Change Theme manages products, baselines, and change requests. Uses Change Authority, Change Budget, and Issue/Change logs.

📙 ISO

ISO 21502 calls for formal change control to evaluate requests and configuration management to ensure product traceability and version integrity.

✅ Similarities

  • All mandate formal evaluation and approval of changes.
  • Configuration management ensures product/baseline integrity.

⚖️ Differences

  • PMBOK = CCB-driven; PRINCE2 = Change Authority/Issue Register; ISO = general governance-level guidance.
